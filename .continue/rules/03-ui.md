# UI / XAML-regler

## Grundregler
- No logic in code-behind — all logic in ViewModel
- No x:Name on controls used only for logic — use bindings
- Never bind directly to parent ViewModel from UserControl

## Bindings
- Input fields: Mode=TwoWay
- Commands: Command="{Binding DeleteCommand}" with CommandParameter
- Design-time: d:DataContext="{d:DesignInstance Type=vm:OrderListViewModelDesign, IsDesignTimeCreatable=True}"

## Styles
- All styles and templates in ResourceDictionary — never inline
- Shared styles in App.xaml or separate ResourceDictionary files
- Control-specific styles in the control's own ResourceDictionary

## UserControls
- Always expose DependencyProperties for data
- Never reference parent ViewModel directly
public static readonly DependencyProperty TitleProperty =
    DependencyProperty.Register(nameof(Title), typeof(string), typeof(MyControl));

public string Title
{
    get => (string)GetValue(TitleProperty);
    set => SetValue(TitleProperty, value);
}

## Converters
- Always implement IValueConverter
- Register in ResourceDictionary — never instantiate inline
- Name: [What]Converter ex: BoolToVisibilityConverter
