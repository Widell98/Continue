# UI / XAML-regler

## Grundregler
- Ingen logik i code-behind — all logik i ViewModel
- Ingen `x:Name` på kontroller som bara används för logik — använd bindings
- Bind aldrig direkt till parent ViewModel från UserControl

## Bindings
- Inputfält: alltid `Mode=TwoWay`
- Commands: `Command="{Binding DeleteCommand}"` med `CommandParameter`
- Design-time:
```xml
d:DataContext="{d:DesignInstance Type=vm:OrderListViewModelDesign, IsDesignTimeCreatable=True}"
```

## Styles och templates
- Alla styles och templates i ResourceDictionary — aldrig inline
- Delade styles i `App.xaml` eller separata ResourceDictionary-filer
- Kontroll-specifika styles i kontrollens egna ResourceDictionary

## UserControls
Exponera alltid DependencyProperties för data — referera aldrig parent ViewModel direkt:

```csharp
public static readonly DependencyProperty TitleProperty =
    DependencyProperty.Register(nameof(Title), typeof(string), typeof(MyControl));

public string Title
{
    get => (string)GetValue(TitleProperty);
    set => SetValue(TitleProperty, value);
}
```

## Converters
- Implementera alltid `IValueConverter`
- Registrera i ResourceDictionary — instansiera aldrig inline
- Namngivning: `[What]Converter` ex: `BoolToVisibilityConverter`
